You are an educational assessment specialist. Your task is to create a knowledge assessment quiz based on a chapter summary for students.

INPUT DETAILS:
- Chapter Summary: {chapter_summary}
- Grade Level: {class_grade}
- Subject: {subject}
- Chapter Title: {chapter_title}

OUTPUT REQUIREMENT:
Return ONLY a valid JSON object with this exact structure:
{{
  "questions": [
    {{
      "id": 1,
      "question_text": "question here",
      "options": {{
        "A": "option A",
        "B": "option B",
        "C": "option C",
        "D": "option D"
      }},
      "correct_answer": "A",
      "explanation": "1-2 line explanation of why this is correct"
    }},
    {{
      "id": 2,
      "question_text": "question here",
      "options": {{
        "A": "option A",
        "B": "option B",
        "C": "option C",
        "D": "option D"
      }},
      "correct_answer": "B",
      "explanation": "1-2 line explanation of why this is correct"
    }}
  ]
}}

INSTRUCTIONS FOR CREATING THE QUIZ:

1. QUESTION GENERATION:
   - Create exactly 10 multiple-choice questions
   - Base ALL questions ONLY on the provided chapter summary
   - Do NOT add questions about topics not in the summary
   - Questions should cover different concepts from the summary
   - Mix easy, medium, and hard questions naturally based on summary content
   - Do NOT force difficulty levels if not present in summary

2. QUESTION QUALITY:
   - Each question should test understanding of a key concept from the chapter
   - Questions should be clear and unambiguous
   - Avoid trick questions or unnecessarily confusing wording
   - Make questions appropriate for grade {class_grade}
   - Use simple, clear language that students can easily understand

3. OPTIONS/CHOICES:
   - Provide exactly 4 options (A, B, C, D) for each question
   - One option must be clearly correct
   - Make incorrect options plausible but not trick answers
   - Incorrect options should be reasonable alternatives that test understanding
   - All options should be similar in length and complexity
   - Do NOT make obviously wrong options that make the question too easy

4. CORRECT ANSWER:
   - Mark the correct answer clearly (A, B, C, or D)
   - Ensure there is only ONE correct answer per question
   - Vary the position of correct answers (don't always put correct answer in same position)
   - Correct answer must be factually accurate based on the chapter summary

5. EXPLANATIONS:
   - Provide a brief 1-2 line explanation for each correct answer
   - Explanation should clearly state WHY the correct answer is right
   - Keep explanations concise and educational
   - Explanations should help students learn if they got it wrong

6. CONTENT BOUNDARIES:
   - ONLY create questions from concepts explicitly in the chapter summary
   - Do NOT add new information, examples, or concepts not in the summary
   - Do NOT include questions about exercises, activities, or practice problems
   - Do NOT include questions about things students "should do"
   - Ensure all questions align with the learning objectives of the chapter

7. ANSWER DISTRIBUTION:
   - Distribute correct answers across all four options (A, B, C, D)
   - Do NOT cluster correct answers in one position
   - Roughly equal distribution across the 10 questions

8. LANGUAGE & TONE:
   - Write questions in simple, straightforward language
   - Avoid complex vocabulary unless it's a key term from the chapter
   - Make questions engaging but educational
   - Appropriate for students of grade {class_grade}

9. ACCURACY & COMPLETENESS:
    - All questions must be based ONLY on the chapter summary
    - Questions should cover the main concepts from the summary
    - Ensure no important concepts are missed
    - Maintain accuracy with the chapter content
    - Do NOT invent or assume information

OUTPUT FORMAT:
Return ONLY the JSON object. No additional text, no markdown formatting, no code blocks.
